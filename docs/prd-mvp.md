## 一、文档概述

### 1.1 编写目的

本文档定义 ParkHub 智慧停车管理平台 MVP 版本的功能需求、交互规范与验收标准，供设计、开发、测试团队参考执行。

### 1.2 产品定位

ParkHub 是面向物业公司与商业综合体的一站式停车管理 SaaS 平台，MVP 版本聚焦"**车辆入场 → 停车 → 缴费 → 出场**"核心业务闭环，支持多租户多车场架构。

### 1.3 核心价值主张

| 用户角色 | 核心价值 |
|----------|----------|
| 物业公司 | 零硬件投入快速上线、多车场统一管理、收入透明可查 |
| 车场操作员 | 异常处理便捷、实时监控直观、远程操作可控 |
| 车主 | 无感入场、自助缴费、快速离场 |

## 二、用户角色与权限

### 2.1 角色定义

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| **平台管理员** | ParkHub 运营方 | 全平台租户管理、系统配置、数据统计 |
| **租户管理员** | 物业公司管理员 | 本租户车场管理、设备管理、计费规则、数据查看 |
| **操作员** | 车场现场人员 | 实时监控、出入记录、手动抬杆、手动收费、异常处理 |
| **车主** | 临停车主（H5） | 扫码查看费用、在线支付 |

| 停车场管理 | ✅ 只读 | ✅ CRUD | ❌ | ❌ |
| 设备管理 | ✅ 只读 | ✅ CRUD | ✅ 远程操作 | ❌ |

#### 3.1.1 页面定位
- **访问角色**：所有后台用户
- **入口场景**：未登录状态访问任意后台页面自动跳转

```
│  │ 数据展示卡片×3      │   │  │ 表单区域            │   │
│  │ 底部版权            │   │  │ 第三方登录          │   │
│  └─────────────────────┘   │  └─────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

#### 3.1.3 组件规格

| 组件ID | 组件名称 | 类型 | 规格说明 |
|--------|----------|------|----------|
| L-001 | Tab切换 | 导航 | 「账号登录」「手机号登录」两个Tab，点击切换表单内容 |
| L-002 | 账号输入框 | 输入 | placeholder: "请输入账号或邮箱"，前置图标：用户 |
| L-003 | 密码输入框 | 输入 | 密码模式，右侧带「显示/隐藏」切换按钮 |
| L-004 | 记住登录 | 复选框 | 默认选中 |

| L-007 | 手机号输入框 | 输入 | placeholder: "请输入手机号"，前置图标：手机 |
| L-008 | 验证码输入框 | 输入 | 宽度自适应，配合「获取验证码」按钮 |
| L-009 | 获取验证码按钮 | 按钮 | 点击后60s倒计时，禁用状态显示"N秒后重发" |
| L-010 | 第三方登录 | 按钮组 | 微信、支付宝图标按钮，MVP暂不实现功能 |

#### 3.1.4 交互行为

**登录按钮点击流程：**
```
1. 前端校验
   ├─ 账号为空 → 输入框标红，提示"请输入账号"
   ├─ 密码为空 → 输入框标红，提示"请输入密码"
   └─ 校验通过 → 进入步骤2

2. 按钮状态变更
   ├─ 显示loading动画 + "登录中..."
   └─ 禁用按钮，防止重复提交

3. 调用登录接口
   ├─ 成功 → 存储 Token，根据角色跳转对应首页
   │         平台管理员 → /tenant-management
   │         租户管理员/操作员 → /realtime-monitor
   └─ 失败 → 按钮恢复，显示错误提示 Toast
```

#### 3.1.5 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 账号不存在 | Toast: "账号不存在" |
| 密码错误 | Toast: "密码错误，请重试" |
| 账号被冻结 | Toast: "账号已被冻结，请联系管理员" |
| 网络超时 | Toast: "网络异常，请稍后重试" |
| 连续错误5次 | 账号锁定15分钟，Toast: "登录失败次数过多，请15分钟后重试" |

```
┌─────────────────────────────────────┐
│ 公司名称 *      [________________]  │
│ 信用代码        [________________]  │
│ 联系人 *        [________] 电话 * [________] │
│ 管理员邮箱      [________________]  │
│ 备注            [________________]  │
│                 [________________]  │
├─────────────────────────────────────┤
│              [取消] [确认创建]      │
└─────────────────────────────────────┘

提交校验：
- 公司名称：必填，2-50字符
- 联系人：必填，2-20字符
- 联系电话：必填，11位手机号格式校验
- 邮箱：选填，邮箱格式校验

- 关闭弹窗
- 刷新列表
```

**冻结/解冻操作：**

```

解冻流程：
1. 点击「解冻」按钮
2. 弹出二次确认： "确认解冻该租户？"
3. 确认 → 调用API → 成功后状态变更为「正常」
```

## 四、业务规则

### 4.1 计费规则

```
输入参数：
- free_minutes: 免费时长（分钟）
- price_per_hour: 计费单价（元/小时）
- daily_cap: 每日封顶（元）

计算逻辑：
1. 停车时长 = 出场时间 - 入场时间（精确到分钟）
2. 计费时长 = max(0, 停车时长 - 免费时长)
3. 计费小时数 = ceil(计费时长 / 60)  // 向上取整
4. 原始费用 = 计费小时数 × price_per_hour
5. 天数 = ceil(停车时长 / 1440)
6. 最终费用 = min(原始费用, daily_cap × 天数)

示例：
- 停车65分钟，免费15分钟，单价8元/小时
- 计费时长 = 65 - 15 = 50分钟
- 计费小时数 = ceil(50/60) = 1小时
- 费用 = 1 × 8 = 8元
```

### 4.2 异常场景处理规则

| 异常类型 | 触发条件 | 系统处理 | 操作员处理 |
|----------|----------|----------|------------|
| 识别失败 | 一体机上报识别失败 | 创建异常记录，车牌标记为"?????" | 手动补录车牌 |
| 有入无出 | 入场后24h未出场 | 标记异常，推送通知 | 确认出场或标记异常 |
| 有出无入 | 出场无匹配入场记录 | 创建异常记录 | 手动补录入场时间 |
| 重复入场 | 同一车牌已在场 | 拒绝入场，通知操作员 | 确认后放行 |
| 设备离线 | 心跳超时 > 60s | 标记离线，推送告警 | 检查设备/网络 |

### 4.3 状态流转

**入场记录状态：**
```
正常入场 → 在场 → 正常出场（已缴费/免费）
                 → 异常（有入无出）
```

**出场记录状态：**
```
正常出场 → 待缴费 → 已缴费
        → 异常（有出无入）→ 已处理
```

## 五、数据规格

### 5.1 核心实体

**租户（Tenant）**
```
{
  id: string,
  company_name: string,        // 公司名称
  credit_code: string,         // 统一社会信用代码
  contact_name: string,        // 联系人
  contact_phone: string,       // 联系电话
  admin_email: string,         // 管理员邮箱
  status: 'active' | 'frozen', // 状态
  created_at: datetime,
  updated_at: datetime
}
```

**停车场（ParkingLot）**
```
{
  id: string,
  tenant_id: string,
  name: string,                // 车场名称
  address: string,             // 地址
  total_spaces: number,        // 总车位数
  available_spaces: number,    // 剩余车位（实时计算）
  status: 'active' | 'inactive',
  created_at: datetime
}
```

**出入口（Gate）**
```
{
  id: string,
  parking_lot_id: string,
  name: string,                // 名称，如"1号入口"
  type: 'entry' | 'exit',      // 类型
  device_id: string,           // 绑定设备ID
  created_at: datetime
}
```

**设备（Device）**
```
{
  id: string,
  serial_number: string,       // 序列号
  parking_lot_id: string,
  gate_id: string,
  type: 'integrated' | 'camera' | 'gate',
  status: 'online' | 'offline',
  last_heartbeat: datetime,
  created_at: datetime
}
```

**通行记录（Record）**
```
{
  id: string,
  tenant_id: string,
  parking_lot_id: string,
  gate_id: string,
  plate_number: string,        // 车牌号
  type: 'entry' | 'exit',      // 类型
  image_url: string,           // 抓拍图片
  timestamp: datetime,         // 通行时间
  matched_record_id: string,   // 匹配的入场/出场记录ID
  fee: number,                 // 费用（出场时）
  status: 'normal' | 'exception',
  exception_type: string,      // 异常类型
  created_at: datetime
}
```

### 5.2 数据隔离

- 所有查询接口必须携带 `tenant_id` 过滤条件
- 平台管理员接口返回全部数据，租户/操作员接口仅返回本租户数据
- 跨租户访问返回 403 Forbidden
