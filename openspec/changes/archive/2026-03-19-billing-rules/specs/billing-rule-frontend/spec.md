## ADDED Requirements

### Requirement: 停车场选择列表
系统 SHALL 在计费规则页面左侧展示当前租户的停车场卡片列表，点击选中后加载对应规则。

#### Scenario: 租户管理员查看列表
- **WHEN** 租户管理员访问计费规则页面
- **THEN** 左侧显示本租户所有停车场卡片
- **AND** 默认选中第一个停车场
- **AND** 右侧加载该停车场的计费规则

#### Scenario: 平台管理员选择租户
- **WHEN** 平台管理员访问计费规则页面
- **THEN** 页面头部显示租户选择下拉框
- **AND** 选择租户后左侧加载该租户的停车场列表

#### Scenario: 无停车场空状态
- **WHEN** 当前租户没有任何停车场
- **THEN** 左侧显示"暂无停车场，请先创建"提示
- **AND** 右侧表单区域禁用

### Requirement: 计费规则摘要卡片
系统 SHALL 在右侧面板顶部以 3 张摘要卡片展示当前规则（免费时长、计费单价、每日封顶）。

#### Scenario: 正常展示摘要
- **WHEN** 选中一个停车场
- **THEN** 显示 3 张摘要卡片：免费时长（分钟）、计费单价（元/小时）、每日封顶（元）

#### Scenario: 每日封顶为零
- **WHEN** daily_cap = 0
- **THEN** 每日封顶卡片显示"不封顶"而非"¥0"

### Requirement: 滑块+输入框双向绑定编辑
系统 SHALL 提供滑块和输入框双向绑定的编辑方式来修改计费规则参数。

#### Scenario: 免费时长编辑
- **WHEN** 拖动免费时长滑块（0-120分钟）
- **THEN** 输入框同步更新数值
- **AND** 手动输入数值后滑块同步移动

#### Scenario: 计费单价编辑
- **WHEN** 拖动单价滑块（1-50元，步进0.5）
- **THEN** 输入框同步更新数值

#### Scenario: 每日封顶编辑
- **WHEN** 拖动封顶滑块（0-500元，步进4）
- **THEN** 输入框同步更新数值
- **AND** 值为 0 时显示"不封顶"标签

### Requirement: 保存和重置操作
系统 SHALL 提供保存和重置按钮来管理规则变更。

#### Scenario: 保存成功
- **WHEN** 用户修改规则后点击保存
- **THEN** 调用 PUT API 更新规则
- **AND** 显示成功提示
- **AND** 摘要卡片更新为新值

#### Scenario: 重置表单
- **WHEN** 用户点击重置按钮
- **THEN** 表单恢复到上次保存的值
- **AND** 丢弃所有未保存的修改

### Requirement: 费用计算器弹窗
系统 SHALL 提供费用计算器弹窗，调用后端 API 模拟计算停车费用。

#### Scenario: 打开计算器
- **WHEN** 用户点击 Header 区域的费用计算器按钮
- **THEN** 弹出模态窗口
- **AND** 显示停车场选择和入场/出场时间输入

#### Scenario: 计算费用
- **WHEN** 用户选择停车场、输入入场和出场时间并点击计算
- **THEN** 调用 POST /api/v1/billing-rules/calculate
- **AND** 展示详细计费明细（停车时长、免费时长、计费时长、计费小时数、单价、每日封顶、天数、原始费用、最终费用）

#### Scenario: 时间校验
- **WHEN** 出场时间早于或等于入场时间
- **THEN** 前端校验拦截并提示错误，不发送 API 请求

### Requirement: 操作员无权访问
系统 SHALL 阻止操作员角色访问计费规则页面。

#### Scenario: 操作员被拒绝
- **WHEN** 操作员角色尝试访问 /billing-rules
- **THEN** 中间件重定向到授权页面
