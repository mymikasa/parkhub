## ADDED Requirements

### Requirement: 页面顶部展示汇总统计卡片
系统 SHALL 在停车场管理页面顶部展示四个统计卡片：总车位数、剩余车位、在场车辆、出入口数。

#### Scenario: 正常显示统计数据
- **WHEN** 租户管理员访问停车场管理页面
- **THEN** 页面顶部显示四个统计卡片
- **AND** 总车位数显示本租户所有车场车位总和
- **AND** 剩余车位显示本租户所有车场剩余车位总和
- **AND** 在场车辆 = 总车位数 - 剩余车位
- **AND** 出入口数显示本租户所有出入口总数

#### Scenario: 数值使用千位分隔符
- **WHEN** 统计数值超过 1000
- **THEN** 显示格式为"8,640"而非"8640"

#### Scenario: 页面加载时显示骨架屏
- **WHEN** 统计数据正在加载中
- **THEN** 显示骨架屏动画
- **AND** 数据加载完成后替换为实际数值

### Requirement: 统计数据 API 响应时间
系统 SHALL 确保统计数据 API 的 P95 响应时间小于 300ms。

#### Scenario: API 性能达标
- **WHEN** 调用 GET /api/v1/parking-lots/stats
- **THEN** 响应时间 P95 < 300ms
- **AND** 返回正确的聚合数据

### Requirement: 统计数据实时性
系统 SHALL 确保剩余车位数据延迟小于 5 秒。

#### Scenario: 数据时效性
- **WHEN** 车辆入场或出场后 5 秒内
- **THEN** 剩余车位数据已更新
- **AND** 统计卡片显示最新数值
